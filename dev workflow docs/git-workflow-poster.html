Git Workflow
Cheat Sheet

Everything you need to commit, push, and stay sane — built for your daily workflow.

The 5 commands to memorize first
01 git status See what changed. Run this constantly.
02 git add . Stage ALL changes for the next save.
03 git commit -m "" Save a named snapshot locally.
04 git push Upload commits to GitHub.
05 git pull Download newest changes from GitHub.

⟳ The Core Loop (repeat every session)

📝 Edit files Work on your novel
→
🔍 git status Check what changed
→
📦 git add . Stage everything
→
💾 git commit -m "msg" Save snapshot
→
☁️ git push Sync to GitHub
$ git status
$ git add .
$ git commit -m "Updated forest scene assets"
$ git push
# or all in one line:
$ git add . && git commit -m "message" && git push

🧠 Git's 3 Zones

Working Dir
Your actual files. Where you edit and delete things.
Staging Area
Changes you've approved with git add. Ready to commit.
Commit History
Saved snapshots. Live in the hidden .git/ folder.

Critical: git add does NOT upload. git commit does NOT upload. Only git push uploads.

📅 Ideal Daily Routine

When starting work
$ git pull
# get latest first
When finishing work
$ git add .
$ git commit -m "msg"
$ git push

Always git pull before you start. Always git push when you're done.

📋 Full Command Reference

# Check what changed
$ git status

# Stage all changes (edits + deletes + new)
$ git add .

# Stage only edits + deletes (no new files)
$ git add -u

# Save snapshot locally
$ git commit -m "Your message here"

# Upload to GitHub
$ git push

# Download latest from GitHub
$ git pull

🔧 Useful Extra Commands

# See all past commits
$ git log --oneline
# press Q to exit

# See exact line-by-line changes
$ git diff

# Check linked GitHub repo
$ git remote -v

# Clone repo to your PC
$ git clone <url>

# Unstage files (undo git add)
$ git restore --staged .

# Undo file changes (permanent!)
$ git restore filename.txt

⚠ Common Mistakes & Fixes

Mistake 1
Forgot to stage before committing
You ran git commit but Git says "no changes added to commit"
→
Fix
Run git add . first, then git commit -m "msg".
If you deleted files specifically, use git add -u instead.
Mistake 2
Committed but GitHub didn't update
You can see your commit locally but the GitHub repo looks old
→
Fix
You forgot git push. Commit only saves locally. Always push after committing.
Mistake 3
fatal: not a git repository
Git Bash doesn't know where your project is
→
Fix
Right-click inside your project folder → "Open Git Bash Here". You must be inside the folder.
Mistake 4
Deleted files not tracked
"Changes not staged for commit: deleted: file.mp4" — Git sees the deletion but won't include it
→
Fix
git add -u stages all deletions & edits.
Or git add . stages everything including new files too.
Mistake 5
Scary wall of terminal text
Git printed a lot of output and you panicked
→
Fix
Read it slowly. Most Git output is just informational. Look for the word "error" or "fatal" — if neither appears, you're fine.

✏ Commit Message Guide

Your commit message is the label on your save point. Make it descriptive.

GOOD ✓
Added intro sequence Finished chapter 1 dialogue Updated UI sprites Fixed branching bug in route A Removed unused recording files Added forest background assets
BAD ✗
stuff changes update aaa fixed it new

Future you will thank present you for good messages.

🌸 Why Git is Perfect for Visual Novels

  • Track every dialogue revision — compare old vs new script versions with git diff
  • Safely experiment with story branches without fear of losing your work
  • Restore any old version of a scene with git log --oneline to find the commit
  • Never lose your work — GitHub is your cloud backup
  • Track asset changes: sprites, backgrounds, music — all versioned
  • Commit often. One scene, one commit. Build a clear timeline.

⚙ One-Time Setup (done once)

# Set your name
$ git config --global user.name "YourName"

# Set your email (same as GitHub)
$ git config --global user.email "you@email.com"

# Verify it worked
$ git config --list

# Check the folder is a repo
$ git status
# should say "On branch main"

💡 What Even Is Git Bash?

  • Git = the version control program. Tracks your files.
  • Git Bash = a terminal window for Windows. It's just how you type commands to Git. Think of it as the keyboard that talks to Git.
  • CLI = any text-based command window. Git Bash, PowerShell, Terminal — all CLIs.
  • GitHub = a website. Completely separate from Git. Just where you store your repo online.
  • Right-click your project folder → "Open Git Bash Here" to always start in the right place.
